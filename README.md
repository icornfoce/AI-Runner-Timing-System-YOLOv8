# 🏃‍♂️ AI Runner Timing System (YOLOv8 + Face Recognition + EasyOCR)

ระบบจับเวลาวิ่งอัตโนมัติด้วย AI สมบูรณ์แบบที่นำเทคโนโลยี **YOLOv8**, **Face Recognition** และ **EasyOCR** มารวมกัน เพื่อตรวจจับใบหน้า ระบุตัวตน และอ่านหมายเลขป้าย (Bib Number) พร้อมระบบตรวจสอบการทำผิดกฎ และสถาปัตยกรรมแบบ Client-Server

## ✨ ฟีเจอร์หลัก
- **YOLOv8 Face Detection:** ตรวจจับใบหน้าได้อย่างรวดเร็วและแม่นยำ 
- **Face & Bib Matching:** ระบุตัวตนจากใบหน้า และตรวจสอบกับเบอร์วิ่ง (Bib) ด้วย OCR
- **Anti-Cheating System:** แจ้งเตือนเมื่อมีการสวม Bib ไม่ตรงกับชื่อ หรือใช้ Bib ของคนอื่น
- **Client-Server Architecture:** รองรับการเชื่อมต่อกล้องหลายจุด (Client) ส่งข้อมูลมารวมที่ Server กลาง
- **Real-time Web Dashboard (Socket.IO):** แสดงผลสถิติและแจ้งเตือนคนทำผิดกฎทันทีโดยไม่ต้องรีเฟรชหน้าเว็บ
- **Judge Panel:** ระบบแผงควบคุมสำหรับกรรมการ พร้อม PIN Code สำหรับอนุมัติ/ยกเลิก (Confirm/Dismiss) ความผิดปกติ

---

## 📁 โครงสร้างโปรเจกต์ใหม่
- `config.py`: ไฟล์ตั้งค่าส่วนกลาง (พอร์ต, IP, Cooldown, PIN)
- `main.py`: โปรแกรมกล้อง AI (เลือก Mode เป็น Client หรือ Standalone ได้)
- `web_app.py`: ตัวเซิร์ฟเวอร์ Flask (Dashboard และระบบ API)
- `client_sender.py`: โมดูลสื่อสารข้อมูลจากกล้องไปยังเซิร์ฟเวอร์
- `helpers/`: โฟลเดอร์รวมฟังก์ชันช่วยเหลือต่างๆ
- `capture_faces.py`: สคริปต์ถ่ายภาพเพื่อลงทะเบียน
- `violations/`: โฟลเดอร์เก็บรูปภาพหลักฐานการทำผิดกฎ

## 📸 วิธีใช้งาน

ระบบรองรับการทำงาน 2 โหมด ซึ่งสามารถเข้าไปปรับเปลี่ยนได้ที่ไฟล์ `config.py`

### โหมดที่ 1: การรันแบบเครื่องเดียว (Standalone Mode)
เครื่องเดียวรันทั้งกล้องและเว็บ Server
1. แก้ไขไฟล์ `config.py` เป็น `MODE = "standalone"`
2. เปิด Terminal ที่ 1 รันกล้อง AI:
   ```bash
   py main.py
   ```
3. เปิด Terminal ที่ 2 รันระบบเว็บไซต์:
   ```bash
   py web_app.py
   ```
4. เปิดเบราว์เซอร์ไปที่ `http://localhost:5000`

### โหมดที่ 2: การรันแบบหลายจุด (Client-Server Mode)
สำหรับงานวิ่งจริงที่มีหลายจุด (สตาร์ท, เส้นชัย, จุดกลับตัว)

**ที่เครื่องเซิร์ฟเวอร์กลาง:**
1. รันระบบเว็บไซต์ (เพื่อให้เป็นจุดรับข้อมูลจากกล้องทุกตัว):
   ```bash
   py web_app.py
   ```
2. สังเกตหน้าจอ Console จะมี IP ของเซิร์ฟเวอร์ เช่น `http://192.168.1.100:5000`

**ที่คอมพิวเตอร์ประจำจุดกล้อง (Clients):**
1. เข้าไปที่ไฟล์ `config.py`
2. ตั้งค่า `MODE = "client"`
3. นำ IP ของเซิร์ฟเวอร์มาใส่ที่ `SERVER_URL = "http://192.168.1.100:5000"`
4. ตั้งค่าหมายเลขจุดผ่าน `CHECKPOINT_ID`
5. รันกล้อง AI:
   ```bash
   py main.py
   ```

## 👨‍⚖️ ระบบกรรมการตรวจสอบ (Judge Panel)
เข้าไปที่หน้าเบราว์เซอร์ `http://localhost:5000/judge` (หรือผ่านลิงก์มุมขวาบนใน Dashboard)
- กรอกรหัส PIN (ค่าเริ่มต้นคือ **`1234`**)
- ตรวจสอบรูปภาพหลักฐานนักวิ่งที่ทำผิดกฎ
- กรรมการสามารถกดปุ่ม **Confirm (ยืนยันว่าผิด)** หรือ **Dismiss (ยกเลิก)** สถานะได้
- สามารถกด Export CSV ออกมาเพื่อสรุปผลการแข่งขันได้

---

## ⚙️ การตั้งค่าระบบ (config.py)
เข้าไปแก้ค่าคงที่ทั้งหมดได้ในไฟล์เดียว:
- `SERVER_URL`: IP ของเซิร์ฟเวอร์กลาง (ในกรณีใช้โหมด Client)
- `MODE`: เลือกระหว่าง "client" หรือ "standalone"
- `CHECKPOINT_ID`: ตัวเลขประจำจุด (เช่น 1=จุดปล่อยตัว, 2=เส้นชัย)
- `JUDGE_PIN`: รหัสผ่านสำหรับระบบกรรมการ
- `COOLDOWN_SECONDS`: เวลาหน่วงป้องกันกล้องบันทึกผลคนเดิมซ้ำจุดเดิม
- `VIOLATION_COOLDOWN_SECONDS`: ป้องกันการถ่ายภาพความผิดปกติซ้ำซ้อนรัวๆ
